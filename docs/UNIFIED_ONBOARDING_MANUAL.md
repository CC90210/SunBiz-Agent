# Unified Onboarding Manual

This is the plain-English handoff the Sun Biz client should experience after pairing the Command Center.

## 1. Meet Solara

Solara is the primary digital employee for Sun Biz Funding.

She helps the team:

- receive new leads from JotForm
- keep follow-up moving
- organize applications and offers
- track funded deals and renewals
- surface what needs attention next

Suga Sean still supports outreach behind the scenes, but Solara is the main face of the Command Center.

## 2. How to work with Solara

The client should be able to use plain English.

Good examples:

- "Show me today's hottest leads."
- "Who still needs follow-up?"
- "Draft a text for this merchant."
- "What is stuck right now?"

The dashboard should keep the technical stack in the background. Client-facing UI should say:

- `Command Center`
- `Local Brain`
- `Automation`

Not:

- `Turso`
- `Dispatch`
- `Substrate`

## 3. What healthy looks like

After credentials are saved, the operator should see a pulse check that confirms:

- Solara is connected to JotForm
- Solara can use Text Torrent for SMS follow-up
- Solara's Local Brain is ready
- the hosted automation layer is online

The ideal chat opener is:

`Hello Jordan, I'm Solara. I've successfully connected to your JotForm and I'm ready to begin processing your funding pipeline.`

## 4. When to loop in CC

Loop in CC when:

- the client needs a workflow changed
- a data mapping looks wrong
- a credential check fails and the operator is not sure why
- the client wants a new automaton or reporting view added

## 5. Pause and correct

If Solara is acting on stale or incorrect information:

1. Pause the task.
2. Correct the source detail in plain English.
3. Ask Solara to confirm the new plan before continuing.

The product should feel calm, guided, and premium from the first screen onward.
